# /data/_source — seed inputs for the build

This folder holds reference data the build prompt uses as a starting point. Your
coding agent reads it when generating the synthetic data set into `/data`.

- **`states.json`** — all 50 US states (`code`, `name`, `region`, `population`).
  The geography entity in `PRD.md` §9.4. The agent copies/normalises this into
  `/data/states.json` and fills any missing field.

Everything else (`buyers.json`, `publishers.json`, `deals.json`) is generated by
the build — see `BUILD-PROMPT.md`. Leave this folder in place; don't edit it by
hand unless you want to change the geography the deals target.
